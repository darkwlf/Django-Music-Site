تقاط دسترسی در پایتون


سلام امروز میخواهیم درباره نقاط دسترسی (access point) ها در پایتون. صحبت کنیم بعضی موقع ها می خواهیم که دسترسی کلاس های فرزند به اتریبیوت ها (خصوصیات) به کلاس پدر محدود بشه پس اینجا میایم و از access point استفاده می کنیم در پایتون 3 نوع access point داریم :
Public
Private
Protected
در Protected همون جوری که از اسمش معلوم هست محافظت شده است ولی بازم این نوع خصوصیات قابل دسترسی هستند و حالت نمایشی دارند که به برنامه نویس بگویند که باید مواظب باشه و با دقت ازش استفاده کنه برای این که اون خصوصیت رو Protected بکنید کافی قبل اسمش یک underscore بگذارید مثلا name_ در Public هم که مثل همیشه استفاده میشه بدون هیچ چیز خاصی مثلا name و حالا میرسیم به Private:

برای Private کردن یک خصوصیت باید دو تا underscore بگذارید ولی فرقش با Protected اینه که دیگه جنبه نمایشی نداره به مثال پایین توجه کنید:


class Student:
__name = 'python' # Private
def _iscrazy(self): # Protected
if self.__name == 'PHP':
return True
return False
age = 20 # Public
class John(Student):
pass
j = John()
print(j.age)
print(j._iscrazy() )
print(j.__name) # We got an error because it is private
#output
20
False
AttributeError: 'John' object has no attribute '__name'


همان طور که در کد بالا مشاهده میکنید ما نمیتونیم از کلاس child به Private ها دسترسی داشته باشیم حال با خود می گویید اگر یک نمونه از کلاس پدر بسازیم می توانیم به خصوصیات Private آن دسترسی داشته باشیم ولی این طور نیست! مانند مثال بالا کد را دوباره می نویسیم فقط نمونه و کار هایی که با کلاس فرزند میکردیم را با کلاس پدر می کنیم :

در مثال بالا متوجه شدیم که ما نمی توانیم حتی از کلاس پدر به خصوصیت Private آن دسترسی داشته باشیم ولی باید توجه داشته باشیم که در پایتون هیچ چیزی زوری نیست و ما باز هم می توانیم از این خصوصیات در کلاس پدر استفاده کنیم برای این کار کافی است که مانند این الگو عمل کنیم classname__identifier_ پس برای کد بالا می توانیم اینگونه به name__
